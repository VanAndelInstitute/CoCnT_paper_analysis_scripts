# CoCnT Analysis Scripts

This repository collects the analysis scripts used to process and analyze the PBMC CoCnT dataset (TODO) and bone marrow CoCnT dataset. The code is organized by analysis stage: each workflow directory contains scripts for one part of preprocessing, donor deconvolution, ArchR analysis, or pseudo-bulk track generation.

Detailed instructions for each stage are documented in the `README.md` file within the corresponding subdirectory.

## Repository Layout

- About sequencing data demultiplexing and mapping, please refering to this workflow: https://github.com/vari-bbc/sciCT_pipeline

- [`run_BM_create_arrow/`](run_BM_create_arrow/)
  Generate ArchR Arrow files from mono-occupancy and co-occupancy BED inputs.
- [`run_BM_souporcell/`](run_BM_souporcell/)
  Prepare per-sample BAM files, export a barcode whitelist, and run Souporcell for donor deconvolution. Since this step need to use `BAM` file as input, and the `BAM` files are very big, the souporcell step will only provide the script and output. But if you are using CoCnT and want to run SouporCells, please check out our workflow here: https://github.com/vari-bbc/souper-star
- [`run_BM_archR_analysis/`](run_BM_archR_analysis/)
  Perform the core ArchR-based analysis, including project creation, QC, clustering, annotation, chromatin-state analysis, and pseudotime comparisons.
- [`run_BM_bed_profile/`](run_BM_bed_profile/)
  Generate fragment QC summaries and pseudo-bulk BED/BigWig tracks by cluster and histone-mark combination.
- [`lib/R/`](lib/R/)
  Helper R functions sourced by multiple analysis scripts.
- [`data/`](./data)
  The fold containing the raw input, it is not included in this repo, but can find in the Zenodo.

## Reproduce the result

For a full re-run, the directories are typically used in the following order:

1. [`run_BM_create_arrow/`](run_BM_create_arrow/)
   Create ArchR Arrow files from raw BED inputs.
3. [`run_BM_archR_analysis/`](run_BM_archR_analysis/)
   Build ArchR projects and run downstream analyses.
4. [`run_BM_bed_profile/`](run_BM_bed_profile/)
   Produce fragment QC summaries and pseudo-bulk coverage tracks.

`run_BM_bed_profile/run_qc.R` can be used earlier for fragment-level QC, but the downstream pseudo-bulk workflow depends on outputs generated by `run_BM_archR_analysis/`.

## What Is In This Repository

This checkout is primarily a script repository. It contains:

- R scripts and shell scripts for each analysis stage
- helper R code under `lib/R/`
- generated outputs in some workflow directories (for example, Arrow files, intermediate `tmp/` files, and figures)
- per-directory documentation describing inputs, outputs, and execution order

The repository does not provide a single workflow manager, package manifest, or containerized environment for end-to-end execution.

## External Inputs and Software

The scripts depend on external data and references not fully included in this repository. Depending on the workflow stage, these will be available in the ./data folder on Zenodo.

- sample sheets for the four bone marrow batches
- raw BED and BAM files
- `hg38` reference genome files
- UCSC genome size files
- Arrow files or the raw inputs required to regenerate them

Most scripts use hard-coded relative paths and write derived files into local `tmp/` directories. Before rerunning any stage, check the corresponding subdirectory README and the top of the relevant script to confirm the expected input paths and output locations.

## Notes for Reuse

- The codebase is script-centric and reflects the order in which the analysis was developed.
- In `run_BM_archR_analysis/`, script numbering is historical; gaps in numbering do not imply missing required steps.
- The per-directory `README.md` files contain the workflow-specific details needed to rerun or adapt each step.
